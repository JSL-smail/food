本项目是一套面向企业内部知识沉淀与智能问答场景构建的 SpringBoot 企业级 AI 知识库平台。系统主要应用于企业制度查询、设备说明检索、生产流程指导、售后知识支持、内部培训答疑等业务场景，通过将传统文档资料与大模型能力结合，实现“自然语言提问—智能知识检索—精准生成回答”的一站式 AI 服务，解决企业内部知识分散、人工查询效率低、员工学习成本高以及客服答复不统一等问题。

在整体架构上，平台采用前后端分离与微服务架构设计，前端支持 Vue 管理后台、H5、小程序及 App 多端接入，通过 Nginx 实现统一负载转发，并由 Spring Cloud Gateway 作为系统统一网关完成接口鉴权、访问限流、请求日志追踪及统一路由分发。后端核心以 SpringBoot + SpringCloud Alibaba 为基础搭建多个独立业务服务，包括用户权限认证服务、AI 对话核心服务、知识库管理服务、会话记录服务、Prompt 模板服务、文档解析切片服务、Embedding 向量化服务、检索召回服务及 Rerank 重排服务等，形成完整的 AI 问答处理链路。

在知识处理层面，平台支持 PDF、Word、Excel、TXT、Markdown 等多格式文档上传，通过文档解析切片服务对原始资料进行文本抽取、结构清洗、语义切块和元数据封装，再调用 Embedding 向量化服务将文本内容转换为高维语义向量，并统一存储至 Milvus 向量数据库中，形成企业专属知识向量索引。当用户发起提问时，AI 对话核心服务首先将问题进行向量化，并通过检索召回服务在 Milvus 中进行 TopK 相似知识召回，再由 Rerank 重排服务对候选知识进行精排筛选，确保返回最相关的知识片段，最后结合 Prompt 模板服务组装上下文提示词，通过 LLM 模型代理服务统一调用 Ollama 或 vLLM 部署的大语言模型完成最终智能回答生成。

在技术选型方面，系统业务数据采用 MySQL 存储用户、权限、知识库及业务配置等结构化信息，聊天记录及上下文历史采用 MongoDB 存储大文本会话数据，热点检索结果及登录令牌使用 Redis 实现高速缓存，文档原文件统一存储至 MinIO，异步任务通过 Kafka 或 RabbitMQ 解耦处理文档向量化流程，定时任务采用 XXL-JOB 完成索引重建与知识同步，底层模型服务与向量数据库均通过 Docker 容器化部署，具备良好的横向扩展能力与企业级可维护性。

总体而言，本项目通过“微服务架构 + RAG知识增强检索 + 大模型智能生成”三位一体的技术融合，构建了一套可私有化部署、可持续扩展、可深度定制的企业智能知识中台，为企业实现知识数字化、问答自动化和办公智能化提供了完整解决方案。
